- Имена всех позиций должны быть уникальными
- Имена направлений из каждой позиции должны быть уникальными
  - Возможно определение направления переводящее позицию саму в себя (вращение)
- Топология доски после конфигурирования не меняется
  - Для изменения (например телепортов), создаётся новая доска (в следующей фазе игры) и переносятся фигуры
- IGrid упрощает конфигурирование, формируя createPosition и addLink
- addSymmetry задаёт несимметричное отношение (обратное отношение можно задать явно)
- Отношение addSynonym не симметрично
  - без флага isGate объявляет позиции тождественными
  - с флагом isGate создаёт "телепорт"
  - допускаются связи n:1, но не 1:n (эмулируется переконфигурированием доски)
- addOperation конфигурирует одновременный перенос нескольких фигур
  - Вместе с фигурами переносятся атрибуты полей (например, признак того что на поле можно размещать фигуру для Platform Chess)
  - Выполнение операции можно связать с перемещением фигуры-рукоятки из опроеделённой позиции в заданном направлении (возможно в другом grid-е)
- При наличии нескольких ходов с совпадающими начальной и конечной позицией выводится popup для выбора хода
- Оптимизировать хранение в State мелких объектов
- Использовать "копирование при записи" для хранения версий состояния фигур
- Реализовать Zobrist hashing на уровне State
- Позиционные флаги - это не клонируемые атрибуты позиций
