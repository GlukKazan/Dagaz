- Генератор ходов
  - configure
    << obj - XML-узел (is-a развёртывается XSLT-преобразованием)
  - generate
    < state - текущее состояние
    >> (old, dsg, new) - (состояние, нотация хода, новое состояние) [1]
    >> (name, dsg) - (позиция, нотация хода) [2]
- Хранилище состояний
  - configure
    < game/players - XML-узел (содержит расстановку фигур, включая резерв)
  - add_state - сохранение сгенерированных состояний [1]
    < old - текущее состояние
    < dsg - нотация хода (уникальная в пределах old)
    < new - новое состояние
  - del_state - удаление состояния (нарушаемые инварианты и отложенные проверки)
    < old - текущее состояние
    < dsg - нотация хода (уникальная в пределах old)
  - get_state - получение нового состояния по нотации хода
    < old - текущее состояние
    < dsg - нотация хода
    > new - новое состояние
  - set_state - переход к новому состоянию
    < state - фиксация состояния (незафиксированные состояния удаляются)
  - get_parent - получение предыдущего состояния
    < state - текущее состояние
    > parent - предыдущее состояние
  - get_turn
    < state - текущее состояние
    > player - текущий игрок
    > turn_number - номер хода
    > turn_order - индекс внутри хода
    > turn_part - номер частичного хода
  - copy_state - создание копии состояния
    < old - текущее состояние
    > new - состояние с изменённым turn_part
  - switch_order - переключение хода
    < old - текущее состояние
    > new - состояние с изменённым (turn_number/turn_order) turn_part = 0
  - get_value
    < state/pos/piece - позиция, фигура или доска
    < name - имя значения
    > value - текущее значение (неизменяемые значения в state не хранятся)
  - set_value
    < state/pos/piece - позиция, фигура или доска
    < name - имя значения
    < is_permanent - признак постоянного значения
    < value - текущее значение (булевские значения группировать в маску)
  - get_position
    < state - текущее состояние
    < name - имя позиции
    > pos - позиция
  - get_piece
    < pos - позиция
    > piece - фигура (null - если позиция пуста)
  - get_player
    < piece
    > player
  - get_type
    < piece
    > type
  - set_piece
    < pos - позиция
    < piece - фигура (null - если позиция пуста)
  - create_piece
    < pos - позиция
    < type
    < player
- Управляющий модуль
  - add_pos_move - получение вариантов ходов [2]
    < name - имя позиции
    < dsg - нотация хода
- Модуль визуализации (плагин)
  - configure
    << board/player/piece - XML-узлы
  - add_piece
    < pos - позиция (или имя зоны резерва)
    < player - имя игрока
    < type - тип фигуры
  - move_piece
    < old - старая позиция
    < new - новая позиция
  - change_value (может использоваться для визуализации превращённых фигур)
    < pos - позиция
    < name - имя значения
    < value - значение
